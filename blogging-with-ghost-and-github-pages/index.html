Blogging with Ghost and Github pages

20 October 2014

I decided to under go the feat of creating a blog with my explots at turing. I initially decide to do blogger since i had already had a google account. But after a very short amount of time i realized that it was not going to allow me to do as much as would like to. So i did some research and found ghost, a very nice and easy to use blogging service that had all the add-ons and custom themes i could ever want.

I loved the ghost platform but i did not like the fact that i had to pay $10 a month to run a blog. When people call me a geek i say thank you so i looked in to free hosting and blogging platforms. I found that ghost could be hosted on a server be it local or web and went on to looking for a way to host it on github pages. I found 2 blogs that explained how the process went but neither of them worked right for me.

So i need to preface the instructions with im running a mac so they maybe slightly different depending on the rig you use.

Start by downloading the enviroments

  • Install Node
    Download Node
  • Install Python
    Download Python 2.7.8
  • Run code it your terminal to install "pip"
    curl --silent --show-error --retry 5 https://bootstrap.pypa.io/get-pip.py | sudo python2.7
  • Then use pip to install buster
    pip install buster
  • Then wget which is required by buster with
    brew install wget

Okay so now that we have everything installed that we need to run the blog lets get the fun part started.

Download the ghost platform to run locally.

  • Download Ghost
  • Unzip and go to the directory (unzip to the directory you want to have ghost live forever)
  • Again in your terminal run to install the server
    npm install
  • And then to run the server locally run
    npm start
  • Visit http://localhost:2368/ to see that everything is working correctly.

So now that we have the blog running locally we will get it set up on github with buster.

Now to create a Github page for your blog

  • Go to you github profile and create a public repo with the name of your account(for me is the repo name was larsonkonr.github.io) but it you really want to i created a seperate account with the name of what i wanted the blog to be.
  • In the same directory that you put the ghost files create a folder with the name of your blog to keep the generated static files separately
  • cd into the folder which you just created
  • then to setup the location for the repo
    buster setup
  • and when promted enter the repo you just created for me that is https://github.com/larsonkonr/larsonkonr.github.io.git
  • once that is completed run this to generate the static files
    buster generate --domain=http://127.0.0.1:2368
  • then cd into static
    cd static
  • Now to get it up on github
    git add .
    git commit -m 'Initial commit'
    git push

All we have to do now is wait. although github says that you only have to wait 10 minutes i had to wait 26 minutes but after everything it was the hardest wait with constantly clicking refresh.

To change any of the styling you have to edit them in the ghost directory. Then boot up your local server and add or change any post you would like. Then to get that on your hosted blog you need to run buster generate --domain=http://127.0.0.1:2368 and then add, commit, and push to github.

But there you have it a blog using ghost hosted on github.
