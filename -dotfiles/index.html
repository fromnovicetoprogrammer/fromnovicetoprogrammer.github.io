.Dotfiles

11 December 2014

If you're a developer or just a techie all the specialized shortcuts and installed packages you have are very important to your flow and productivity and if you heaven forbid happen to lose your computer it would be extremely hard to reproduce everything you had, or if you are using multiple computers consistency between machines is key. With this tutorial you will be able to save and retrieve those precious files with ease.

Lets start by checking out all of the dotfiles

  • Open your terminal and navigate to your root directory
  • Enter ls-a and it will give you all the files even the hidden ones which we are looking for.
  • Take note of all the ones you would like to save i.e. vim, bash, gitignore and any others there you would like to save.
  • Create a directory with in root name .dotfiles
    mkdir .dotfiles
  • Then we can start to move your files into that directory ( you can either keep the files as dots or you can change them to something viewable
    mv .vimrc .dotfiles/vimrc etc for all the files and folder you want
  • When all files are transferred we will symlink all the files back to where they came from. Go to the .dotfile directory and run this for every file
    ln -s vimrc ~/.vimrc
  • Now that we have everything symlinked all that is left is to get them saved on github
    git init
    git add .
    git commit git push

Now what ever the situation all you need to do is pull the repo and symlink everything back and you will be back to peak productivity.
